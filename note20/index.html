笔记20-线性基(最大XOR) 「专题」

发表于 2018-11-05 |

线性基，XOR的艺术。
来源/网址：luogu

线性基

背景

众所周知,XOR是一种神奇的操作。其中最重要的一条操作是:

若A XOR B == C 则A XOR C==B

我们现在有若干个数可以无限次XOR（事实上，使用两次就相当于没有使用，所以每个数最多用一次）。那么这些数能否操作后，得到的最大值是什么呢？

这时候，就轮到线性基出场了。 线性基通过将原数组转换为等价（值域完全相同）但最高位数依次降低的数组来实现这一操作

操作与原理

首先，我们需要一个数组 p[MAX] 其中MAX为原数组中二进制最高位数

依次扫描每一个数。将其转化为二进制（通过位运算）从最高(非0)位开始往最低位扫。假设当前扫到数x的第j位，若当前位为0，那么直接continue；如果该数该位是1，再次进行分类讨论：

若p[j]为0，则将该数赋值给p[j]，并直接break。很显然，这个操作不影响最后XOR得到的数的值域。

若p[j]不是0，那么就进行x^=p[j]，继续下一个循环。这个操作的原理，就是背景中的那个重要公式。 x^p[j]得到的数再 ^p[j]依然可以得到x。而根据上一种情况，显然也能得到p[j]（在二进制下）最高非0位就是j的结论。所以进行这一部操作后，我们达到了降低x(在二进制下的)最高位的同时，不改变值域的目的。

最终得到的数组p最高位依次下降，即为线性基。

例题一

洛谷P3812 模板题

有了最高位依次下降后，不管查询最大值或是是否可以XOR得到某一值都十分简单。 大致就是从高往低扫，能 增大ans / 将操作数该（二进制）位转成与所求数相同 则进行操作，最后直接输出ans / 判断是否相同（其实中间就可以直接break) 即可。

例题二

洛谷P4151

你有一张无向图（有重边、自环），每条边上都有一个值。你要从1走到n，中途路径可以重复，求经过所有边上的值XOR起来，最大是多少

思路

首先从1到n随便走一条路径，得到XOR的值ans。

然后找出图中所有环的XOR值，放到数组e中。对数组e进行线性基操作，得到线性基数组p

最后以原路径ans为底，通过线性基p求最大值。

至于找所有换其实很简单，从1开始dfs,并记录vis数组。若已经vis过该点i，则代表成环了。但注意：要预先记录dis（第一次搜索到该点时的xor)，并将成环时的xor与dis[i]再次进行XOR，得到的便是该环的XOR (XOR两次相当于没有XOR)。
